# Sprint 1 14-Day Activation Plan

Sprint: 1  
Window: 14 days  
Capacity target: 32 strategic hours total (16 hours/week)  
Execution split: 50% Revenue / 50% Load Reduction

---

## Operating Rules

- Stay inside Sprint 1 deliverables only.
- All off-sprint ideas go to `idea_parking_lot.md`.
- No national, multi-city, or SaaS/platform work.
- Each day ends with tracker updates in `S1_EXECUTION_TRACKER.md`.

## Daily Cadence (Default)

- 90-minute deep-work block #1
- 90-minute deep-work block #2
- 20-minute QA and documentation update

---

## Day-by-Day Execution

### Day 1 - Revenue

- Populate `S1_CASE_STUDY_01.md` with real project data.
- Attach visual asset links and verify premium ($10K+) framing.
- Update status and blockers in tracker.

### Day 2 - Revenue

- Populate `S1_CASE_STUDY_02.md`.
- Standardize outcome and investment language.
- Update tracker.

### Day 3 - Revenue

- Populate `S1_CASE_STUDY_03.md`.
- Final QA across all 3 case studies for consistency.
- Update tracker.

### Day 4 - Revenue

- Complete core copy inside `S1_PORTLAND_PREMIUM_LANDING_PAGE.md`.
- Insert case-study references and CTA flow.
- Confirm Portland-only language.

### Day 5 - Revenue

- Build publish-ready page from landing scaffold in your web stack.
- Run a final content QA checklist and mark page as live once published.
- Update exit-gate row for "Landing page live."

### Day 6 - Revenue

- Configure campaign structure from `S1_GOOGLE_ADS_SETUP_PACK.md`.
- Build ad groups and keyword sets in draft mode.
- Add baseline negatives and conversion mapping.

### Day 7 - Revenue QA

- Complete ad copy variants and destination mapping.
- Perform full pre-launch QA; keep campaign not live unless approved.
- Update exit-gate row for "Ads configured and ready."

### Day 8 - Load Reduction

- Deploy `S1_CLIENT_REQUEST_INTAKE_SYSTEM.md` as your mandatory intake process.
- Communicate enforcement rule to all request handlers.
- Start logging every request ID.

### Day 9 - Load Reduction

- Run first 5-10 real intake submissions through triage tags.
- Resolve any ambiguity in Included/Change Order/New Proposal logic.
- Update exit-gate row for "Intake system enforced."

### Day 10 - Load Reduction

- Populate `S1_PROPOSAL_MODULAR_TEMPLATE.md` with real modules and terms.
- Define tier defaults and exclusions.
- Perform a dry run on one recent opportunity.

### Day 11 - Load Reduction QA

- Run 2 timed proposal assembly tests.
- Tighten template until average build time is under 30 minutes.
- Update exit-gate row for proposal readiness.

### Day 12 - Load Reduction

- Populate `S1_PRODUCTION_PLAYBOOK_SKELETON.md` with operational detail.
- Assign owner roles for each checklist section.
- Publish playbook v1 to team.

### Day 13 - Integration

- Cross-check links and handoffs:
  - Landing CTA -> Intake system
  - Intake outcomes -> Proposal flow
  - Proposal scope -> Playbook planning
- Fix friction points and document process notes.

### Day 14 - Gate Readiness Review

- Perform full Sprint Exit Gate review in `S1_EXECUTION_TRACKER.md`.
- Mark each requirement Ready/Not Ready with evidence.
- Log any non-sprint ideas to `idea_parking_lot.md`.
- Define Sprint 2 carryovers only for unfinished Sprint 1 items.
